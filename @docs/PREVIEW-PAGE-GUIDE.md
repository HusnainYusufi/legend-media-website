## ğŸ¬ Animation Reference

### Hero
- **Logo**: Fade in + scale (0.9 â†’ 1) on load
- **Tagline Cycle**: Fade out/in with slide up/down (Think â†’ Analyze â†’ Create)
- **CTAs**: Fade in with stagger delay
- **Scroll Indicator**: Bounce animation (optional)

### About Section
- **Heading**: Fade in + slide up when in view
- **Description**: Fade in with delay
- **Pillar Cards**: Stagger animation (0.1s delay between each)
- **Cards on Hover**: Subtle lift (translateY -8px) + shadow increase

### Stats Section
- **Number Cards**: Fade in + slide up when in view
- **Counter Animation**: Count from 0 to target value (1.5s duration)
- **Stagger**: 0.1s delay between cards

### Projects Section
- **Project Cards**: Fade in + slide up when in view
- **Images**: Lazy load with fade in
- **Card on Hover**: Lift effect (translateY -8px) + shadow
- **Badge Animation**: Subtle pop-in with stagger

### Testimonials Section
- **Carousel**: Infinite horizontal scroll (auto-play)
- **Speed**: Slow, smooth scrolling (no jank)
- **Pause on Hover**: Stop animation when hovering
- **Cards**: Fade in as they enter viewport

### CTA Section
- **Background**: Subtle gradient animation (optional)
- **Button**: Magnetic effect on hover (follow cursor)
- **Text**: Fade in + slide up

### General Principles
- **Timing**: 300-600ms for most animations
- **Easing**: Spring physics or easeOut for natural feel
- **Threshold**: Trigger 100px before element enters viewport
- **Once**: Most animations trigger once (don't repeat on scroll)

## ğŸ’¡ Design Tips

1. **White Space**: Don't be afraid of generous padding
2. **Alignment**: Keep everything grid-aligned
3. **Consistency**: Reuse spacing values, don't create new ones
4. **Hierarchy**: Clear visual hierarchy with size, weight, color
